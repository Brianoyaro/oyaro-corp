Build Springboot application
cd backend
./mvnw clean package -DskipTests


Create a dedicated Linux user if absent
sudo adduser brian

Move the jar file
sudo mkdir -p /opt/<brian-folder>
sudo cp target/*.jar   /opt/<brian-folder>/app.jar
sudo  chown brian:brian /opt/<brian-folder>

Make springboot persistent using systemd
sudo vim /etc/systemd/system/<application_name_or_any_name.service   e.g student-management.service>
**paste this code in the file**
[Unit]
Description=Student Management System Backend
After=network.target

[Service]
User=springboot
WorkingDirectory=/opt/student-management

ExecStart=/usr/bin/java -jar /opt/student-management/app.jar

SuccessExitStatus=143

Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
**code end**


Reload
sudo systemctl daemon-reload


Enable Startup
sudo systemctl enable student-management


Start service
sudo systemctl start student-management


check status
sudo systemctl status student-management


view logs
journalctl -u student-management -f



Configuring a new reverse proxy - called ? in our case. Replace ? with actual file name
sudo nano /etc/nginx/sites-available/<? replace this with actual file name>

server {
    server_name api.example.com;

    location / {
        proxy_pass http://localhost:<PORT e.g 8080>;
        
        client_max_body_size 50M;

        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Enable site
sudo ln -s /etc/nginx/sites-available/<? change this to actual file name> /etc/nginx/sites-enabled/

Test nginx configuration
sudo nginx -t

Install certbot
sudo apt install certbot python3-certbot-nginx -y

Generate certificate
sudo certbot --nginx -d <api.example.com>



Configure SpringBoot for production.
